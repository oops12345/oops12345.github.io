计算机端口

  • 端口的作用是区分服务

  • 端口是唯一的，不可以重复使用

  • 端口号范围1-65535

  • 根据端口可以探测 host的开放的服务、操作系统、角色等

  • SSH、SCP、端口重定向 ：22

  • TFTP (Trivial File Transfer Protocol)：69/udp
  • SMTP (simple Mail transfer Protocol) : 35/tcp

  • TOMCAT :8080
  • win2003: 3389–RDP
  • Oracle ：1521
  • MS sql*server ：1433/tcp 1433/udp
  • mysql :3306
  • qq :1080/udp

注册表

注册表是win中一个核心数据库，存放各种参数

win+D—–regedit

注册表结构

  • HKEY_CLASSES_ROOT

    管理文件系统，根据Windows中安装的应用程序的扩展名，该根键指明其文件类型的名称，相应打开文件调用的程序等等信息

  • HKEY_CURRENT_USER

    管理系统当前的用户信息 ，这个根键中包含了本地计算机中存放的当前登录的用户信息，，包括用户登录用户名和暂存的密码，win98时，其信息是由HKEY_USERS相应的项拷贝过去的

  • HKEY_LOCAL_MACHINE

    • 管理当前系统的硬件配置，这个根键中保存了本地计算机硬件中的配置数据，此根键下的关键字包含在SYSYTEM.DAT中，用来提供HKEY_LOCAL_MACHINE所需的信息，或者在远程计算机可访问的一组键中，

    • 这个根键中许多子键与system.ini文件中的设置类似

  • HKEY_USERS

    管理系统用户信息，

  • HKEY_CURRENT_CONFIG

    管理当前用户的系统配置，这个根键中保存当前用户的桌面配置，

注册表的使用
  • 查询开机启动程序
  • 克隆账号的密码权限
  • 读取服务密码
  • 隐藏后门
常用DOS命令

  • ping

    ping 简介：源站点向目的站点发送ICMP request报文，目的主机收到会icmp repaly报文 ，验证两个接点之间的可达性

    功能：用ping 来判断两个连接点在网络层的连通性

    • ping的参数

      • ping -n ： 连续ping n 个包
      • ping -t 持续的ping 直到认为中断，ctrl+break暂时中止ping命令
      • ping -l 指定ping 报文所携带的数据部分的字节

    • ping的错误原因

      • unknow host ：主机不可以被解析为ip地址，故障原因可能是DNS server 。

      • Network unreachable ：表示本地系统没有到达远程地址的路由。 检查路由表的配置

      • NO answer ：表示本地系统有到远程主机的路由，但是没有接收到返回的报文

      • Request timeout : 可能是远程主机禁止了icmp报文或者硬件链接问题

    • arp -d 在arp缓存中删除一条

    arp -d 127.0.0.1

    • arp -g 显示所有表项

  • tracert 追踪路由

    tracert是为了探测源节点到目标节点之间数据报文经过的路径。

    利用ip报文的ttl域在每个经过一个路由器转发后减一，如果此时ttl=0 则向源节点报告超时这个特性，从一开始逐一增加ttl，直到到达目标节点或者ttl到达最大值255

    功能 ：探索两个节点之间的路由

    常用参数 ：

    • tracert ip_address

    • tracert -h n (设置ttl最大值为n)

  • netstat

    显示协议统计信息和TCP/IP连接 ，该命令只有安装了TCP/IP协议才能用

    参数

    • -a 显示 所有连接和侦听端口，服务器连接通常不显示
    • -e 显示以太网统计 该参数可以与-s结合使用
    • -s 显示每个协议的统计默认情况下显示 TCP UDP ICMP和ip的统计
    • -p protocol 可以指定显示的子集

    • -n 以数字形式显示ip地址和端口号 （而不是尝试查找名称）

    • -r 显示路由表的名称

    • -interval 重新显示所有统计 ，在每次显示之间暂停interval秒 ctrl + B停止 重新显示 统计

    • 如果省略该参数 netstat 将打印一遍当前的配置信息

  • route

    路由是ip层的核心问题，路由表是tcp/ip协议栈所必须的核心数据结构，是ip选路的唯一依据

    功能 ： route是操作，维护路由表的重要工具

    常用参数 ：

    route print 查看本机路由表

    route add 增加一条路由记录

    route add 1.1.0.0 mask 255.255.0.0 10.110.41.20 metric 3

    永久添加一条路由记录

    route -p add 1.1.0.0 mask 255.255.0.0 10.110.41.20 metric 4

    删除一条路由记录

    route delete 1.1.0.0
