博客搭建终极版（Hexo+Github+PicGo+腾讯云COS）

搭建博客的初衷是写写平时遇到的一些问题及解决问题的方法，对学到的新东西进行记录并分享出来，但是尝试过种种方法，包括但不限于hexo、hugo、 typecho等各种博客系统以及各种笔记软件 ，但是都不舒服，这一套流程总是感觉有问题，最最醉后，这算是目前为止最终极的解决方法，当然每个人对于类似问题的解决模式不一样，保持分享的冲动，慢慢就会找到最适合自己的一套模式 。

这一套流程主要是本地typora写东西 ，hexo通过git推送到网页端，图片问题主要有自建图床，下载安装typora，功能强大的编辑器，hexo的部分主要有hexo本体安装，GitHub端设置，域名解析问题和最终的美化问题，

Hexo本体的安装

必要组件

首先是结合官网以及各个博客主的（简称博主）的教程，首先是安装必要的组件

下列组件具有时效性，尽量安装最新的稳定版

  • Git（下载链接）

    百度百科：Git（读音为/gɪt/）是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

    我们的博客内容的上传和更新也就是通过git进行版本控制，Git下载完成一路next，到最后一步注意选择Use Git from the Windows Command Prompt，自动添加环境变量，即可以直接在Cmd（命令提示符）使用，最后通过git --version检查安装是否成功

  • node.js（下载链接）

    Hexo是高效的静态站点生成框架，它基于Node.js，下载完成安装过程一路继续next，最后在打开win+R打开Cmd（命令提示符），通过node -v和npm -v检查是否安装成功。

hexo本地文件安装（win为例）

hexo的本地安装也分为全局安装和局部安装

  • 全局安装

    首先在喜欢的合适位置创建一个文件夹（命令类似hexo/blog等），定位到该文件夹下，单击鼠标右键，选择Git bash here输入命令npm i hexo-cli -g进行hexo的全局安装，最后通过hexo -v检查安装的版本

  • 局部安装（初始化）

    • hexo init hexo的初始化

    tips: 错误console.js:39 throw new ERR_CONSOLE_WRITABLE_STREAM('stdout'); 由于node.js版本过低，去官网升级版本即可

    • hexo g 本地静态页面的生成
    • hexo s本地服务器开启，通过访问localhost的4000端口查看服务是否开启成功

至此，hexo的文件本地基本安装结束。

hexo的主题安装（themes/butterfly）

hexo 5.0版本的主题安装

npm i hexo-theme-butterfly
